Apa Fungsi Agent Ini?

DIL Autonomous Ebook Agent adalah sistem agent yang mengorkestrasi workflow pembuatan ebook teknis premium:

  • Memory Agent — mengingat konteks dan histori.
  • Task Planner — membuat rencana tugas dan estimasi biaya.
  • Router Agent — memilih provider AI yang tersedia.
  • Outline Agent — membuat outline bab dan subbab.
  • Writer Agent — menulis konten dengan 5 lapisan wajib: [KONSEP], [ANALOGI], [RUMUS], [CONTOH], [APLIKASI].
  • Reviewer & Repair Agent — memvalidasi dan memperbaiki konten.

Output utama berupa ebook.md yang kemudian diubah menjadi website statis dan PDF (best-effort).

Output Artifact

Setelah workflow berjalan, artifact yang dihasilkan antara lain:

  • ebook.md Konten utama ebook.
  • site/index.html Website statis untuk preview.
  • site/style.css Gaya website.
  • ebook.pdf PDF jika pandoc tersedia.
  • ebook_pdf_fallback.txt Jika pandoc belum tersedia.
  • run_report.json Laporan eksekusi.
  • cost_report.json Laporan biaya.
  • website_validation_report.json Validasi website.
  • pdf_validation_report.json Validasi PDF.

Cara Menjalankan via GitHub Actions

Mode Test (Tanpa API)

Workflow test_export.yml dapat dijalankan secara manual untuk memverifikasi export website dan PDF fallback tanpa memerlukan API key.

  1. Buka tab Actions di repository.
  2. Pilih workflow Test Export.
  3. Klik Run workflow → Run workflow.
  4. Tunggu hingga selesai dan unduh artifact.

Mode Produksi

Workflow agent_run.yml memerlukan GitHub Secrets berupa API key. Pastikan secret sudah dikonfigurasi sebelum menjalankan.

Deploy Website

Workflow deploy_site.yml dapat dijalankan manual untuk membangun dan menerbitkan site/ ke GitHub Pages.

  1. Aktifkan GitHub Pages di repository Settings (source: GitHub Actions).
  2. Jalankan workflow Deploy Site dari tab Actions.
  3. Akses website di URL GitHub Pages repository.

Catatan Keamanan

  • Jangan menaruh API key, token, atau secret di file repository mana pun.
  • Jangan mengaktifkan auto-posting tanpa approval.
  • Jangan membuat klaim penghasilan pasti atau gratis tanpa limit.
  • Semua API memiliki kuota dan biaya masing-masing.
  • Fallback writer tersedia jika API tidak tersedia.

Status: Test Mode First

Repository saat ini beroperasi dalam mode test untuk memvalidasi pipeline export website dan PDF. Mode produksi penuh akan dibuka setelah:

  • test_export.yml berhasil.
  • website_validation_report.json status PASS.
  • PDF ada atau fallback note tersedia.
  • GitHub Pages berhasil menampilkan site.
  • README sudah menjelaskan cara pakai.

Silakan jalankan workflow di Actions untuk memulai.
