Tentang Proyek

DIL Autonomous Ebook Agent adalah kumpulan agent AI yang berjalan di GitHub Actions untuk membantu produksi ebook teknis berbahasa Indonesia. Agent ini menghasilkan draft Markdown, mengonversinya menjadi website statis, dan menyiapkan laporan validasi sebelum konten dipublikasikan.

Halaman ini adalah website publik resmi proyek, dipublikasikan melalui GitHub Pages dari branch main folder /docs.

Status Website: Online Source: main / docs Versi: Awal Produksi

Fungsi Agent

Sistem terdiri dari beberapa agent yang bekerja berurutan:

  • Memory Agent — Mengingat konteks dan histori run.
  • Task Planner — Membuat rencana tugas dan estimasi biaya.
  • Router Agent — Memilih provider AI yang tersedia berdasarkan model_pool.
  • Outline Agent — Membuat outline bab dan subbab.
  • Writer Agent — Menulis konten dengan 5 lapisan: [KONSEP], [ANALOGI], [RUMUS], [CONTOH], [APLIKASI].
  • Reviewer & Repair Agent — Memvalidasi dan memperbaiki konten.
  • Site Builder & Validator — Mengubah Markdown menjadi HTML statis dan memeriksa kelayakannya.

Hasil utama: output/ebook.md → website statis (/docs/ebook.html) + laporan validasi JSON.

Cara Menggunakan Agent

1. Jalankan Lewat GitHub Actions

  1. Buka tab Actions pada repository.
  2. Pilih workflow yang sesuai:
    • Test Export — Validasi pipeline tanpa API key.
    • agent_run — Produksi ebook (memerlukan API key di Secrets).
    • Production Website Check — Memvalidasi folder /docs.
  3. Klik Run workflow.
  4. Setelah selesai, unduh artifact yang dihasilkan.

2. Buat Issue Generate Ebook

Gunakan template issue [EBOOK] Judul Ebook untuk memicu produksi ebook. Workflow akan memproses dan menempelkan laporan di issue.

3. Lihat Preview

Buka /ebook.html untuk melihat contoh hasil render ebook.

API Key & Secrets

Agent membaca API key dari GitHub Actions Secrets. Tidak ada API key yang disimpan di kode atau file repository.

Secret yang dipakai kode saat ini:

  • PROVIDER_1_API_KEY — OpenAI (GPT-4o / GPT-4o-mini)
  • PROVIDER_2_API_KEY — Gemini slot 1
  • PROVIDER_3_API_KEY — Gemini slot 2
  • PROVIDER_4_API_KEY — Gemini slot 3
  • PROVIDER_5_API_KEY — OpenRouter

Tersedia untuk integrasi berikutnya (belum aktif di kode utama):

  • HUGGINGFACE_API_KEY
  • TAVILY_API_KEY
  • PERSONAL_GITHUB_TOKEN (jika dibutuhkan untuk operasi lintas-repo)

Catatan transparansi: nama secret seperti GEMINI_API_KEY atau OPENROUTER_API_KEY tidak dibaca langsung oleh kode. Pemetaannya melalui slot PROVIDER_*_API_KEY sesuai config/model_pool.json.

Catatan Keamanan & Kebijakan

  • Jangan menaruh API key, token, password, atau data rahasia di file repository.
  • Tidak ada auto-posting ke media sosial atau marketplace tanpa approval manusia.
  • Tidak ada klaim "gratis tanpa limit" — semua API memiliki kuota dan biaya masing-masing.
  • Tidak ada klaim penghasilan pasti — output agent adalah alat bantu, bukan mesin uang otomatis.
  • Tidak ada sistem pembayaran yang dipasang di repository ini.
  • Semua publikasi memerlukan review dan approval manusia.

Jika Anda menemukan secret atau data rahasia tertulis di file mana pun di repository, segera laporkan ke pemilik repository dan rotasi (revoke) secret tersebut.
